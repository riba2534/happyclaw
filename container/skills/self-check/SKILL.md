# Self-Check 纠错记忆系统

你拥有一份持久化的纠错清单，记录了过去犯过的每一个被用户纠正的错误。
这份清单是你最重要的学习资料之一——它代表了用户花时间帮你改正的真实教训。

## 纠错清单位置

`/workspace/global/correction-log.md`

这个文件在用户的全局记忆目录中，跨会话持久化。

## 两个核心行为

### 行为一：记录纠错（用户纠正你时）

当用户指出你犯了错误时：

1. **理解错误本质**：不要只记表面现象，要理解为什么会错。比如用户说"SGOV 计价错误，又把份额当金额计算了"，本质是"有价证券的价值 = 份额 × 净值，不能直接用份额数作为价值"。

2. **读取现有清单**：用 Read 工具读取 `/workspace/global/correction-log.md`，检查是否已有类似条目。如果已有，更新该条目（增加出错次数、补充新的触发语境）而不是新增重复记录。

3. **追加新条目**：在清单顶部（`## 纠错记录` 标题之后）插入新条目，格式如下：

```markdown
### [YYYY-MM-DD] 场景标签 - 错误类别
- **错误**: 一句话描述做错了什么
- **正确做法**: 一句话描述应该怎么做
- **根因**: 为什么会犯这个错（帮助举一反三）
- **触发语境**: 用户原话摘要
```

字段说明：
- **场景标签**：这个错误容易在什么场景下出现（如：资产计算、基金查询、数据库操作、价格刷新、文件处理）
- **错误类别**：计算错误 / 数据理解错误 / 遗漏步骤 / API 调用错误 / 逻辑错误 / 格式错误
- **根因**：这是最重要的字段——用一句话解释为什么会犯这个错，方便下次遇到类似场景时快速联想

4. **向用户确认**：简短告知已记录，不需要长篇大论。例如："已记录到纠错清单。"

### 行为二：预防重犯（执行任务前）

在执行以下类型的操作之前，先读取纠错清单中的相关条目：

- 数值计算（尤其是资产价值、份额、净值相关）
- 数据库写入或更新
- 价格/汇率获取和应用
- 创建或修改文件
- 调用外部 API

**具体做法**：
1. 用 Read 工具读取 `/workspace/global/correction-log.md`
2. 扫描场景标签，找到与当前任务相关的条目
3. 在执行前内心复盘这些教训（用 `<internal>` 标签），确保不重蹈覆辙
4. 如果当前操作涉及清单中记录的易错模式，在执行时格外谨慎，必要时多做一步验证

这个预防步骤不需要告诉用户你在做——它应该是无声的、自然的，就像一个经验丰富的人在动手前先回忆一下以前踩过的坑。

## 判断边界

**应该触发记录的**：
- 用户明确指出你做错了（"错了"、"不对"、"有问题"）
- 用户纠正你的输出（"应该是 X，不是 Y"）
- 用户表达你重复犯错（"又来了"、"上次就说过"）
- 用户要求你记住某个错误（"记住这个错误"、"别再犯了"）

**不应该触发的**：
- 用户说"记住这个"但说的不是纠错（如"记住我喜欢喝咖啡"）——这属于偏好记忆，不是纠错
- 用户在讨论别人的错误，不是你的错误
- 用户提出新需求或改变主意（"还是换成 X 吧"不是纠错，是需求变更）
